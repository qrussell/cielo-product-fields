Cielo Custom Product Fields - Folder Layout

cielo-product-fields/
│
├── cielo-product-fields.php       # The core plugin file (Header info & Initialization)
├── package.json                   # NPM config for @wordpress/scripts (React compiler)
│
├── includes/                      # All PHP backend and integration logic
│   ├── admin-meta-box.php         # Bridge 1: Registers CPT meta box & enqueues React
│   ├── class-frontend-render.php  # Bridge 2: Hierarchical Merge Engine (Product Page)
│   ├── class-cart-handler.php     # Bridge 3: Cart validation, pricing, and uploads
│   └── class-license-manager.php  # Bridge 4: API calls to cielocloud.org & Pro checks
│
├── src/                           # React Source Code (Your uncompiled development files)
│   ├── index.js                   # React mount point (ReactDOM.render)
│   ├── TemplateBuilder.jsx        # The drag-and-drop builder UI we created
│   └── builder-styles.scss        # SCSS for the builder interface
│
├── build/                         # Compiled React Assets (Auto-generated by NPM)
│   ├── index.js                   # The minified JS enqueued by admin-meta-box.php
│   ├── index.asset.php            # Auto-generated dependencies array for WP
│   └── style-index.css            # Compiled CSS for the admin builder
│
└── assets/                        # Traditional frontend assets (Loaded on product page)
    ├── js/
    │   └── frontend-fields.js    # Vanilla JS for live price updates (Phase 4)
    └── css/
        └── frontend-fields.css    # Styling for the injected product page inputs


Breakdown of the Layout:

The Root Level: Contains your main cielo-product-fields.php file, which should eventually be kept very lean. Its only job will be to require_once the files inside the includes/ folder and boot up the classes. It also holds your package.json for managing the React build tools.

The includes/ Folder: This is where your PHP lives. As your plugin grows, splitting the logic (Admin vs. Frontend vs. Cart Processing) into separate class files keeps the codebase from turning into spaghetti. We just built admin-meta-box.php for this folder!

The src/ Folder: This is your React playground. You edit TemplateBuilder.jsx here. WordPress never actually reads these files directly.

The build/ Folder: When you run npm run build, your @wordpress/scripts pipeline takes everything in src/, compiles it, minifies it, and spits it out here. This is what admin-meta-box.php actually loads into the browser.

The assets/ Folder: Since we only need React in the backend admin screen, your frontend product pages will use standard, lightweight JavaScript and CSS. Those files go here to keep your store loading fast.
